# Yes, And — A Director's Prompt-Book

**Yes, And** is a single-file, no-build demo of a dual-model improv tool, presented as a **game-film replay**: one hand-authored two-hander scene between a PLAYER and an AI ACTOR, typeset as a real playscript and annotated in the right margin like a director's grease-pencil prompt-book. Press **Play** and the lines type onto the page one at a time; a beat after each line lands, the director's notes scratch in — a see-saw **status** glyph that tips toward whoever is up, a **platform** "established" marker once the base reality is set, the **tilt** (the first unusual thing) circled in oxblood, and the **game** of the scene named as a one-sentence pattern. A scrubber and Step control let you replay the finished scene and watch the notes fire at their stored timestamps, and it ends on a Harold-style notes card grading the scene against the canon with one concrete "next time, try…".

## How to run

### The replay (no server, no key, works offline)

The replay is one self-contained `index.html` with no build step. Either:

- **Open it directly** — double-click `index.html` (or `open index.html` on macOS), **or**
- **Serve it** — `python3 -m http.server` in this folder, then visit `http://localhost:8000/`.

Fonts (Spectral, Caveat, Archivo) load from the Google Fonts CDN, so an internet connection gives the intended typography; everything else is inline. **Opened as a bare `file://`, the replay is fully functional** — Live mode just won't have a proxy to call, and degrades to a graceful inline notice (see below).

**Controls:** `Play / Pause` · `Step` (and the right arrow key) advances beat-by-beat · the **scrubber** seeks anywhere · `Space` toggles play, `←/→` step. The see-saw, the circled tilt, and the Harold card are all live UI driven by a single timeline.

### Live mode (real model play — needs the proxy + an API key)

**Live mode is now real.** Click **● Live mode** and you improvise *against* the model in the same playscript + grease-pencil UI: you type a line, the **AI ACTOR** replies, and the **director** scratches its read (status / platform / tilt / game) into the margin.

It runs through a Vercel Node serverless function, [`api/improv.mjs`](api/improv.mjs), which holds the Anthropic key **server-side** so it never ships to the browser. To run it you need:

1. **An Anthropic API key**, set as the `ANTHROPIC_API_KEY` environment variable.
2. **The `/api/improv` function deployed (or running locally).** Locally:

   ```sh
   npm install                 # installs @anthropic-ai/sdk for the function
   export ANTHROPIC_API_KEY=sk-ant-...
   vercel dev                  # serves index.html + /api/improv together
   ```

   Then open the printed URL and click **● Live mode**. On Vercel, set `ANTHROPIC_API_KEY` in the project's environment variables and deploy — the static `index.html` and the `api/improv.mjs` function ship together, no build step.

If Live mode can't reach the proxy (you opened the bare file, or no key is configured), it shows an inline notice — *"Live mode needs the deployed proxy with ANTHROPIC_API_KEY — replay still works"* — and the hand-authored replay remains fully available via **Exit live**.

**Models used:** the latency-sensitive AI ACTOR runs on `claude-sonnet-4-6`; the quality-sensitive director runs on `claude-opus-4-8`. The large theory-engine and director-rubric system prompts are sent with `cache_control` for prompt caching across turns.

## What's real vs. illustrative

- **The replay (default, hand-authored):** the entire transport — Play, Step, the scrubber, the beat ticks, the see-saw status, the tilt-circle draw-on, the staggered margin notes, and the Harold notes card — is driven from one timeline computed from the scene data, so scrubbing backward and forward reconciles the visible notes correctly. The 12-line scene **and every director note** are written by hand as an exemplar of good improv craft (accept the offer, under-write, build a clear platform, one clean tilt, then find and *heighten* the game). **Nothing in the replay is generated by a model** — that is the honest nature of a replay demo, and it needs no server and no key.
- **Live mode (real, model-generated):** when you switch to Live mode and have the proxy configured, the AI ACTOR's lines and the director's beats *are* generated at runtime by Anthropic models through `api/improv.mjs`. This is the live counterpart to the replay, rendered in the identical UI.

## How live mode is wired

`api/improv.mjs` is a Vercel Node serverless function with two modes, both reading `ANTHROPIC_API_KEY` from the server environment:

- **`mode: 'actor'`** (`claude-sonnet-4-6`, latency-sensitive) — plays the AI ACTOR under a **theory-engine** system prompt encoding the craft constraints: accept offers, under-write (one new gift per line), no platform-stalling questions, establish who/what/where then find and heighten the *game*, and make your partner look good. Returns `{ line, stageDirection }`.
- **`mode: 'director'`** (`claude-opus-4-8`, quality-sensitive) — scores the scene against the canon via a forced tool call, returning strict JSON: `{ status, platform: { established, baseReality }, tilt: { landed, line }, game: { stated, oneLine } }`.

Both system prompts are large and stable, so they're sent with `cache_control: { type: 'ephemeral' }` for prompt caching across turns. If `ANTHROPIC_API_KEY` is missing the function responds `500` with a clear JSON error, which the client surfaces inline. The client (`index.html`) POSTs the running transcript to `/api/improv` and renders both the actor's reply and the director's beats into the same playscript + margin-rail UI used by the replay.

## Theory grounding

The craft vocabulary comes from two canonical sources:

- **Keith Johnstone, *Impro: Improvisation and the Theatre* (1979)** — *status* transactions, building a *platform*, the *tilt* (the first unusual thing that breaks routine), and "make your partner look good."
- **Matt Besser, Ian Roberts & Matt Walsh, *The Upright Citizens Brigade Comedy Improvisation Manual* (2013)** — *base reality*, the *first unusual thing*, and finding/heightening the *game of the scene* (heighten the same unusual thing rather than stacking new ones).

The annotations in the demo are the author's reading of those texts applied to a hand-written scene; they are not quotations.
